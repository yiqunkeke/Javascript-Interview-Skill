## 函数优先

函数声明和变量声明都会被提升。

但是，

### 1. 函数会首先被提升，然后才是变量。

```
	foo();
	
	var foo;  // 重复的 var 声明会被忽略掉！
	
	function foo() {
		console.log(1);
	}
	
	foo = function(){
		console.log(2);
	}
```

会输出1，这个代码片段可以被`引擎`理解为如下形式：

```
	function foo() {
		console.log(1);
	}
	
	foo(); // 1
	
	foo = function() {
		console.log(2);
	}
```

注意，var foo 尽管出现在 function foo()...的声明之前，但它是**重复的var声明(因此被忽略)**。因为函数声明会被提升到普通变量之前。

-----------------------------------------------------

### 2. 重复的 var 声明会被忽略掉。

-------------------------------------------------

### 3. 出现在后面的函数声明还是可以覆盖前面的。

```
	foo(); // 3
	
	function foo() {
		console.log(1);
	}
	
	var foo = function() {
		console.log(2);
	}
	
	function foo() {
		console.log(3);
	}
	
```

虽然这些听起来都时无用的学院理论,但是它说明了在同一个作用域中进行重复定义是非常糟糕的,而且经常导致各种奇怪的问题。

### 4. 尽可能避免在块内部声明函数。
